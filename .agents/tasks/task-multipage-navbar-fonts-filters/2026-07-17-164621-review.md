# Multi-page restructure with shared Navbar, display font, and split filters (v2)

InternHub was a single-page app with anchor-link navigation. This change breaks it into four App Router routes (`/`, `/for-employees`, `/for-students`, `/post`), extracts a shared `Navbar` and `Footer` into `layout.tsx`, adds Playfair Display as a display font for headings, and splits the "Certifications & webinars" filter into two independent checkboxes with OR semantics when both are active. The v1 review flagged five issues — all addressed in the latest fix commit: dark mode now uses `useSyncExternalStore` with a server snapshot, filter logic uses OR when both are checked, the post form shows a demo disclaimer, and footer placeholder links use `#`.

**Watch for:** The Navbar renders a side-effectful `document.documentElement.classList.toggle` call during the render phase (outside useEffect), which works in practice but violates React's expectation that render is pure (confirmed). The `suppressHydrationWarning` on `<html>` suppresses the console warning for the class mismatch but doesn't eliminate the FOUC itself — dark-mode users still see a white flash before `useSyncExternalStore` re-renders on the client (confirmed, inherent to static generation without an inline script).

**Verdict**: APPROVED

## High-level view

The dark mode implementation now uses `useSyncExternalStore` with a server snapshot returning `false`, eliminating the SSR crash risk from the v1 review. The FOUC remains inherent to static generation (server HTML is always light-themed); eliminating it requires an inline `<script>` in `<head>` or middleware-based class injection, both outside the scope of this feature. The DOM class sync happens in the render body rather than an effect, which is technically impure.

The filter logic branches correctly: OR when both checkboxes are active, single-attribute filtering otherwise. The post form now carries a visible demo disclaimer.

## Dark mode: side effect during render

The fix commit correctly adopts `useSyncExternalStore` as the SSR-safe primitive, but the DOM sync moved into the render body:

```tsx
// Keep the DOM class in sync
if (typeof document !== "undefined") {
  document.documentElement.classList.toggle("dark", darkMode);
}
```

This executes every time `Navbar` renders. In React strict mode (dev), components render twice — the class toggles twice per cycle, which is harmless but symptomatic of the real issue. If React interrupts and discards a render (concurrent features, Suspense boundaries), the DOM mutation from the discarded render persists with no matching commit. The component renders unconditionally at the layout level so the practical risk is low, but wrapping the mutation in `useEffect` or `useInsertionEffect` would be strictly correct and future-proof.
